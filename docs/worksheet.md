## 4. 詳細設計（アルゴリズム）

### 4.1 メインループ

> メニュー表示→入力受付→処理分岐→ループの流れを記述してください。
> フローチャートまたは疑似コード（箇条書き）で記述してください。

```
1. メインメニューを表示する
2. ユーザーのIDを受け取る
3. 商品バーコードを通す
4. 泊数を選ぶ

6. 支払い方法選ぶ
   1 現金
      お釣りがあれば返す
   2 クレジット
      分割かどうか
   3 バーコード
   4 交通系IC
7. 完了画面
```

### 4.2 個別機能のアルゴリズム

> 各機能（購入処理、ログ保存、ログ表示など）の処理フローを記述してください。


#### 機能5:   伝票ログ
```
1. 貸出処理完了後、伝票番号を自動生成する
2. ユーザーID、商品バーコード、貸出日時、泊数、支払い方法、合計金額を取得する
3. 取得した情報をCSVファイル形式に変換する
4. 伝票ログとしてCSVファイルへ追記保存する
5. 保存完了後、完了画面を表示する
```
